# 产品路线图 · Roadmap

> 最后更新：2026-03-30

## 当前阶段：MVP 验证（2026 Q1）

- [x] 微信小程序 MVP（周报展示 + 用药确认 + 回声）
- [x] Android Demo App（离线假数据演示）
- [x] 后端 API 基础框架（Express + SQLite）
- [x] HTML 高保真原型（4 个核心页面）

## 近期计划（2026 Q2）

### 🔐 微信登录与后端联调
- 接入真实微信登录（wx.login + code2Session）
- 用户数据持久化（绑定关系、步数、用药记录）
- API 鉴权与安全加固

### 📱 小程序上架准备
- 完善隐私协议与用户协议
- 通过微信小程序审核（内容安全、隐私合规）
- 准备上架素材（截图、描述、类目选择）

### 🧪 内测与反馈
- 招募 5–10 个真实家庭参与内测
- 收集使用反馈：周报内容是否易懂、用药确认流程是否顺畅
- 根据反馈迭代 UI 和交互

### ☁️ 微信云开发 PoC
- 评估微信云开发作为后端替代方案的可行性
- 对比自建后端 vs 云开发的成本和维护复杂度
- 如果可行，迁移部分服务到云开发

## 中期计划（2026 Q3）

### 🏥 功能扩展
- 步数数据接入（微信运动 API 或手动录入）
- 用药计划管理（自定义用药方案）
- 多家庭成员支持（一个子女关注多位老人）

### 📊 数据与洞察
- 周报趋势图（步数、用药依从性随时间变化）
- 异常检测与提醒（连续多天步数骤降等）

### 🌐 多端适配
- 小程序端适配更多机型
- 考虑 H5 版本（非微信用户也能用）

## 长期愿景

- 开放 API，允许第三方集成（养老机构、社区健康服务）
- 探索 B2B 模式（社区、物业、保险公司合作）
- 多语言支持（面向海外华人家庭）
