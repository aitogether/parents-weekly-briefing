# 《父母周报》P0 隐私与数据安全说明（草案）

> **适用范围**：仅针对 P0 阶段（只接入微信运动步数 + 手动用药记录），不涉及小米手环等外设。
> **文件路径**：projects/pain-scanner/aging/privacy-and-security.md

---

## 1. 我们收集哪些数据（P0）

### 子女侧

- 微信账号基础标识（openid/unionid，用于登录和绑定家庭关系）。
- 使用行为：打开周报时间、是否滑到底、是否点击「聊了/没聊」。

### 父母侧

- 微信运动步数（最近 30 天的日总步数，不含精确轨迹和地理位置）。
- 用药记录：由子女或父母在小程序内添加用药计划（药名可脱敏昵称）、剂量、服药时间点，服药是否完成由父母在提醒弹窗中手动点击确认。系统不自动抓取医院/药店数据。

### 不收集的数据（P0 明确写死）

- 不读取任何聊天内容、通话记录、短信内容。
- 不读取微信联系人列表。
- 不采集精确地理位置、GPS 轨迹。
- 不接入银行账户、支付记录、医保消费等金融数据。

---

## 2. 数据从哪儿来、存到哪儿

### 数据来源

- **微信运动步数**：通过微信小程序提供的「微信运动」授权接口获取最近 30 天的日步数，只有在用户授权后才会拉取。
- **用药记录**：由子女或父母在小程序内添加用药计划（药名可用昵称）、剂量和服药时间，父母在每日提醒弹窗中点击"已吃"/"没吃"完成确认。系统不自动抓取医院/药店数据。

### 存储位置

- 所有数据加密存储在国内云服务器，遵守《个人信息保护法》（PIPL）中对敏感个人信息的要求（最小必要、用途限定、存储期限控制）。
- OpenClaw 参与的部分，只接收「周汇总特征」（如：本周总步数、相对上月的增减百分比、用药完成率），不保存可单独识别到某个家庭成员的原始数据。

---

## 3. 数据如何用于生成周报

系统只在以下两个场景使用这些数据：

1. 每周一生成一份「父母周报」；
2. 作为匿名统计，观察整体产品效果（如平均焦虑评分变化）。

用药记录只用于计算完成率和在周报中展示，不会用于医疗决策，也不会共享给任何医院、药店或保险公司。

周报中展示的是「本周总步数」「与上月对比」「用药完成率」这类汇总信息，不展示逐小时步数曲线，不展示任何地理位置信息。

---

## 4. 授权与撤回（微信小程序规范对齐）

初次使用时，小程序会按微信隐私规范弹出「隐私保护指引」，只有在你同意后，才会调用微信运动授权接口和存储用药记录。

你可以随时在「设置 → 账号与隐私」中：

- 关闭微信运动授权（之后我们不再获取新的步数数据）；
- 删除父母的用药记录；
- 注销账号：触发删除与你账号相关的所有可识别数据（在 T+N 天内完成删除）。

---

## 5. 谁可以看到这些数据

| 角色 | 可见范围 |
|------|---------|
| **子女账号** | 只看得到自己绑定的父母的周报，不会看到其他家庭的数据 |
| **父母** | 如需查看，可以由子女在手机上展示周报内容；后续版本可以考虑为父母提供单独查看入口（需另行授权） |
| **内部团队** | 运营、产品、技术人员平时只接触脱敏统计数据（如「步数平均增加 10%」「用药完成率中位数 92%」），不按姓名/手机号查看单个家庭 |
| **第三方** | 不会向任何广告平台或数据公司出售、出租或以其他方式提供你父母的个人数据 |

---

## 6. 安全措施（简要）

### 传输安全

- 小程序与服务器之间的通信使用 HTTPS 加密。

### 存储安全

- 敏感字段（如手机号）使用加密存储；
- 后端访问控制基于最小权限原则（每个服务只接触自己必须的数据）。

### 日志与监控

- 访问日志会记录接口调用频次和错误信息，但不记录完整个人数据；
- 异常访问（如暴力请求）会触发自动告警。

---

## 7. 跨境传输（P0 阶段的承诺）

P0 阶段所有数据处理和存储均在中国境内服务器完成，不进行跨境传输，以避免触发 PIPL 额外的跨境评估要求。

## 附录：工程落地指引

本说明同时作为技术实现的约束文档。工程团队需按以下指引落地：

### 数据库字段设计约束

| 数据类型 | 字段 | 存储要求 |
|---------|------|---------|
| 微信标识 | `wx_openid` / `wx_unionid` | 加密存储，仅用于登录和绑定 |
| 步数 | `daily_steps[]`（30天日步数数组） | 只存汇总值，不存轨迹 |
| 用药 | `med_name`（可脱敏昵称）, `dose`, `schedule`, `taken[]`（每日是否服用） | 父母或子女添加计划，父母在提醒中确认服药，不自动抓取 |
| 行为埋点 | `report_opened_at`, `scrolled_to_bottom`, `feedback_type` | 匿名统计用，不关联个人信息 |

### API 接口约束

- 所有接口必须走 HTTPS。
- 步数获取只调用 `wx.getWeRunData()`，返回的是加密 code，需后端解密。
- 后端解密后的步数数据只保留 30 天滚动窗口，过期自动删除。
- 周报生成 pipeline 只读取汇总特征（总步数、对比值、完成率），不传原始明细到生成层。

### 微信小程序隐私弹窗流程

```
用户首次打开小程序
  → 调用 wx.getPrivacySetting() 检查隐私协议状态
  → 未同意 → 弹出隐私保护指引（按微信规范配置 privacy.json）
  → 用户同意 → 调用微信运动授权 + 开始录入用药
  → 用户拒绝 → 仅展示引导页，不获取任何数据
```

小程序 `privacy.json` 配置要点：
- `privacyPolicyUrl`：指向本说明文档的在线版本
- `usePrivacyCheck`：设为 `true`，强制走隐私检查流程
