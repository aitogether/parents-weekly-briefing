## 第1-2周：L1 核心护城河建设（立即启动）

### 任务1：情感化文案库结构化

**目标**：将600-800条抱怨转化为「场景-话术」数据库

**执行步骤**：
1. **读取抱怨库**
   ```bash
   # 假设抱怨库位置
   data/complaints-raw.txt  # 或 complaints-raw.md
   ```

2. **场景分类（手动或半自动）**
   - 用药焦虑（漏服、自行减量、药物冲突）
   - 步数焦虑（突然下降、长期不动、担心不活动）
   - 失联焦虑（联系不上、手机静音、独居风险）
   - 就医困惑（看不懂报告、流程复杂、陪诊需求）
   - 情绪变化（语气低落、不愿沟通、孤独感）

3. **建立数据结构**
   ```json
   {
     "scene": "用药焦虑",
     "user_quotes": ["原话1", "原话2", ...],
     "insight": "子女无法确认父母是否规范用药...",
     "severity_levels": {
       "温和": {
         "templates": ["模板1", "模板2"],
         "action": ["建议1", "建议2"]
       },
       "轻度": {...},
       "严重": {...}
     }
   }
   ```

4. **产出物**：
   - `data/scene-templates.json`（完整文案库）
   - `docs/wording-guidelines.md`（文案使用指南）

**验收标准**：
- 5个核心场景全部覆盖
- 每个场景≥3个文案模板（温和/轻度/严重）
- 每个场景有明确的「触发条件」定义

---

### 任务2：周报心智统一

**目标**：在所有触点上强化「周报」认知

**执行步骤**：
1. **视觉设计**
   - 设计「第X周 · 父母这一周」标题样式
   - 选择温暖色调（橙黄 #FFD700、浅橙 #FFE4B5）
   - 设计专属图标（时钟+房子+心形组合）

2. **推送节奏**
   - 时间：每周五晚7点（子女下班后，父母睡前）
   - 文案：「这一周结束了，点击查看爸妈的动态 📊」

3. **产品内统一**
   - 所有页面标题带「第X周」
   - 周报封面设计模板
   - 分享卡片带「父母这一周」水印

4. **传播语打磨**
   - 主口号：「爸妈这一周，我每周都看」
   - 辅助语：「不是监控，是关心」
   - App Store/小程序介绍强调「每周简报」

**产出物**：
- `docs/visual-identity.md`（视觉规范）
- `assets/weekly-report-template.png`（封面设计）
- `docs/push-strategy.md`（推送策略）

**验收标准**：
- 所有用户看到的周报有统一视觉
- 推送时间固定，形成习惯
- 用户能说出「这是每周的报告」

---

### 任务3：微信运动集成稳定化

**目标**：确保步数数据获取稳定可靠

**执行步骤**：
1. **权限申请**
   - 在小程序后台申请「微信运动」权限
   - 类目选择：「健康」或「生活服务」
   - 准备说明文档（说明用途：仅用于周报）

2. **数据采集逻辑**
   - 每日定时获取（建议早8点）
   - 异常值过滤（单日>10万步标记为异常）
   - 缺失值处理（连续3天无数据触发提醒）

3. **异常检测规则**
   ```javascript
   // 示例规则
   if (今日步数 < 1000 && 历史平均 > 3000) {
     标记为「活动显著下降」
   }
   if (连续3天步数 < 500) {
     标记为「可能健康问题」
   }
   ```

4. **降级方案**
   - 微信运动不可用时，提示「请检查微信运动权限」
   - 可手动录入步数（临时方案）

**产出物**：
- `miniprogram/utils/wechat-steps.js`（数据采集模块）
- `backend/rules/step-anomaly.js`（异常检测规则）

**验收标准**：
- 数据获取成功率 > 95%
- 异常检测准确率（需内测验证）
- 有明确的降级提示

## 第3-4周：MVP 功能验证

### 任务4：Mock 周报样本生成

**目标**：用模拟数据生成3份不同场景的周报

**场景设计**：
1. **一切正常**：用药完整、步数稳定、有回声
2. **轻度异常**：1-2次漏服、步数下降10%、无回声
3. **严重预警**：多次漏服、步数下降50%、失联

**执行**：
- 在本地环境生成3份 JSON 格式的周报数据
- 用小程序预览功能查看效果
- 导出为 PDF 用于用户测试

**产出物**：
- `test/mock-reports/weekly-report-01.json`（正常）
- `test/mock-reports/weekly-report-02.json`（轻度异常）
- `test/mock-reports/weekly-report-03.json`（严重预警）
- `test/mock-reports/screenshots/`（截图）

### 任务5：用户测试（5个家庭）

**测试问题清单**：

**A. 整体价值**
1. 「这份周报对你有用吗？哪里最打动你？」
2. 「哪部分你跳过不看？为什么？」
3. 「你愿意每周都看吗？什么情况下你会忘记？」

**B. 文案感受**
4. 「这些文案像子女说的话吗？还是像医疗报告？」
5. 「『爸妈这周走了3000步』和『本周步数3000』，你更喜欢哪个？」
6. 「看到『⚠️』会紧张吗？会不会觉得被吓到？」

**C. 行动建议**
7. 「『建议周末打个电话』这个建议，你会照做吗？」
8. 「什么样的建议你一定会采纳？什么样的会忽略？」

**D. 对比竞品**
9. 「如果小米运动也推出子女端，你会换吗？为什么？」
10. 「你愿意为这个服务付费吗？多少钱你觉得值？」

**记录方式**：
- 录制屏幕（用户操作过程）
- 记录用户表情和语气
- 收集书面反馈

**产出物**：
- `test/user-testing/feedback-summary.md`
- `test/user-testing/quotes.md`（用户原话）
- `test/user-testing/improvements.md`（需改进点）

### 任务6：内测准备

**内测招募**：
- 渠道：家庭群、朋友圈、V2EX/知乎（异地养老话题）
- 筛选条件：
  - 80/90后，父母在老家（非同城）
  - 父母年龄60-75岁
  - 父母会用智能手机（至少会扫码）
  - 愿意提供每周反馈

**内测规模**：10个家庭（首批）

**内测周期**：4周（每周发送周报，收集反馈）

**成功指标**：
- 周打开率 > 60%
- 用药确认率 > 70%（如果P0有用药功能）
- 回声使用率 > 30%
- 留存率（4周） > 40%

**产出物**：
- `docs/pilot-manual.md`（内测手册）
- `miniprogram/pages/pilot/`（内测用户入口）

## 第5-8周：P0 版本上线与数据积累

### 任务7：MVP 功能开发与上线

**功能清单**（P0）：
1. 子女端：查看周报（用药+步数+回声）
2. 父母端：用药确认 + 一键回应
3. 微信运动接入
4. 周报生成引擎（基于模板）
5. 数据存储（云开发）

**开发优先级**：
- 第1周：后端API + 数据库设计
- 第2周：子女端周报页 + 父母端确认页
- 第3周：微信运动接入 + 数据同步
- 第4周：联调测试 + 修复Bug

**技术要点**：
- 使用云开发（已选型）降低运维
- 微信运动数据每日定时获取
- 周报生成每周五定时任务

**产出物**：
- 可上线的小程序版本
- API 文档 `api.md`
- 部署脚本 `deploy.sh`

## 第9-12周：L2 护城河扩展

### 任务9：家庭关系链网络（P1）

**功能设计**：
- 一个子女可绑定多个父母（女婿/儿媳视角）
- 一个父母可被多个子女查看（兄弟姐妹共享）
- 角色权限：子女（查看+管理）、父母（确认+回应）

**交互流程**：
1. 子女A 邀请父母绑定（生成二维码）
2. 父母微信扫码同意
3. 子女B 也可通过同一二维码绑定（父母同意后）
4. 周报同时推送给A和B

**壁垒效果**：一旦家庭关系建立，迁移成本极高

## 总结

**护城河的本质不是「别人不能做」，而是「别人不愿做/不敢做」**：

- 巨头不愿做：因为「周报+子女端」市场规模对他们来说太小，投入产出比低
- 新创公司不敢做：因为需要长期积累用户信任和数据
- 你的机会：在巨头看不上、新创不敢做的**缝隙市场**，建立深度护城河

**核心执行原则**：
1. 速度优先：3个月内上线MVP，6个月内获取1000个家庭
2. 深度优先：在「周报」单一场景做到极致体验
3. 情感优先：文案永远面向「焦虑的子女」，不是「健康数据消费者」
4. 数据优先：从第一个用户开始积累，建立飞轮

**记住**：你的产品不是健康监测工具，是「减轻子女焦虑」的情感产品。这是巨头最难复制的部分。
