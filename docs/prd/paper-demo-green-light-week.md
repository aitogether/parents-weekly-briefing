# 纸上 Demo 2：全绿灯周

> **同一角色**：35岁深圳打工，爸妈在汕头乡下，妈高血压，爸爱赌牌
> **本周状态**：妈血压全周正常、用药100%全勤，爸步数恢复到上月水平
> **核心挑战**：没有任何异常时，周报怎么写才不是"一切正常→天气不错多晒太阳"

## Step 2：话题参考——怎么写得不像硬凑

这是 Demo 2 的核心难题。绿灯周的话题参考不能是：

❌ "最近天气不错，多注意休息"——废话
❌ "可以问问爸妈最近吃了什么"——像营养师
❌ "问问爸有没有去散步"——明明步数已经正常了，问什么

**我的尝试**：

```
┌─────────────────────────────────────┐
│                                     │
│  💬 这周随便聊聊就好                 │
│  不用聊正事。以下是你可能想知道      │
│  但平时没空问的：                    │
│                                     │
│  • 爸周六出门了——去哪了？跟谁？      │
│  • 妈的血压连着两周没偏高——          │
│    她最近心情怎么样？                │
│  • 清明刚过，老家扫墓了吗？          │
│    邻居阿伯身体怎么样？              │
│                                     │
└─────────────────────────────────────┘
```

> **自检**：
> - ✅ **"爸周六出门了——去哪了？跟谁？"**——这个是从数据里**自然长出来的**。步数恢复+周六2280步是个具体事件，子女看了会觉得"爸是不是出去打牌了"但周报不说，只呈现事实，让子女自己判断。**比"建议多问问爸最近在忙什么"自然得多。**
> - ✅ **"她最近心情怎么样？"**——从"血压两周没偏高"这个事实引出情绪话题，有因果链，不是硬转。
> - ⚠️ **"清明刚过，老家扫墓了吗？邻居阿伯身体怎么样？"**——这句取决于周报发出的时间是否真的在清明后。如果不是特定节日，就是硬凑。**需要一个"基于时间/节气/事件"的话题池，而不是固定的每周重复。**
> - ⚠️ **"平时没空问的"**——这句话隐含了一个假设："你平时打电话只问正事"。对某些子女来说可能是真的，但对另一些子女来说可能觉得被冒犯。

> - ⚠️ **"随便聊一下"**——又出现了"随便"这个词。子女会不会觉得被系统逼着"随便"？

## 🔍 绿灯周 Demo 的 4 句自检

| # | 原文 | 问题 | 建议改 |
|---|------|------|--------|
| 1 | "晚上随便聊两句就行，不用问身体，聊点别的" | "随便"说多了像反话 | 删掉"随便"，直接说："如果周末有空，可以打个电话，不聊身体，聊点家常。" |
| 2 | "平时没空问的" | 假设子女只聊正事，可能冒犯 | 改为："下面几个话题你可能会感兴趣"——中性，不预设 |
| 3 | "清明刚过，老家扫墓了吗？" | 依赖节气，非通用 | 改为动态话题池：系统根据日期/天气/节气/上次聊天间隔自动生成 |
| 4 | "不用提醒了，这周我放心" | "我放心"是系统帮子女说的 | 改为："不用提醒"——更诚实，不要替用户表达感受 |

## 📌 关键结论：绿灯周的话题参考怎么才自然

**发现：绿灯周的话题参考不需要"聊天建议"，需要的是"从数据里长出来的观察"。**

- ✅ **"爸周六出门了——去哪了？跟谁？"** ← 这是数据事件，不是建议
- ✅ **"她最近心情怎么样？"** ← 从"血压两周稳定"因果引出
- ❌ **"天气不错多晒太阳"** ← 没有数据锚点，是硬凑

**绿灯周话题参考的正确写法**：不给"建议聊什么"，而是给"你可能好奇但没注意的数据点"。周报的绿色不代表"没事了"，而是"数据里有值得你了解的日常细节"。

**建议更新 PRD §6**：绿灯周的话题参考槽位应该叫「你可能好奇的」而非「建议聊什么」——语义不同：前者是"我帮你发现了一个细节"，后者是"你该做点什么"。
