# 纸上 Demo：周一早晨一次完整交互

> **角色**：35岁，深圳打工，爸妈在汕头乡下
> **妈**：高血压，吃药控制中
> **爸**：爱赌牌
> **设备**：妈戴小米手环，爸也戴小米手环
> **数据线**：穿戴设备（活动+心率）+ 用药依从（妈的降压药）

---

## Step 0：周一早上 7:32，手机锁屏通知

```

妈本周整体稳定，爸活动量略降。
点开查看详情 →
```

> **文案检查**：✅ 不是"有新消息"或"您的周报已生成"，直接给结论，锁屏上就能看到关键信息。

> - ✅ **"妈这边好转了，可以放一放"** —— 这句很重要：它告诉子女"你上周的行动有效果"，建立正反馈循环。同时给子女"permission to stop worrying about mom"，减轻认知负担。

## 🔍 我自己觉得"我不会这么说"的 3 句话

| # | 原文 | 问题 | 建议改 |
|---|------|------|--------|
| 1 | "爸连续三天步数不到 1000" | 太像审讯——"爸你干嘛了？" | 改为："爸这周出门比较少，周六只有 412 步"——用事实代替数字，降低审判感 |
| 2 | "下次周报会更准" | P0 技术上做不到，不该承诺 | 改为："你的反馈会帮助我们持续改进"——诚实但保持闭环感 |
| 3 | "不一定是大问题" | 太刻意降级，像在安慰 | 改为："原因可能很多（天气、身体不舒服、就是想休息）"——给出多种可能，让子女自己判断 |

## 📌 Demo 总结

周报的核心体验闭环是：**子女看到数据 → 得到一句话结论 → 获得"建议聊什么"的话术 → 做出行动 → 回到周报看上次行动的结果**。

这个闭环在 Demo 里是成立的。最关键的文案发现是「话题参考」——PRD 里没写，但反向 PRD 第8条（"文案要照顾情绪"）和 D层 TOP10 句子（"我以为懂技术就能保护家人，但我错了"）都在指向它：子女需要的不是"建议你打个电话"，而是"建议你打个电话，**而且你可以这么说**"。

**建议正式把「话题参考」纳入 PRD §6 周报骨架，作为行动建议下方的子槽位。**
