## 一、用户画像（Personas）

###  persona 1：异地子女（主要用户）
- **年龄**：30-50 岁
- **职业**：城市白领/技术工作者
- **特征**：
  - 与父母异地居住（不同城市）
  - 工作繁忙，无法每日联系
  - 担心父母健康和安全
  - 愿意为父母花钱（付费意愿强）
  - 熟悉智能手机和微信
- **目标**：
  - 实时了解父母健康状况
  - 及时获知异常和危险
  - 远程提供帮助和支持
- **痛点**：
  - 打电话父母总说「一切都好」，但实际有问题
  - 无法确认父母是否按时服药
  - 担心父母独自外出安全

---

### persona 2：年迈父母（次要用户）
- **年龄**：60-80 岁
- **特征**：
  - 独居或空巢（子女不在身边）
  - 有慢性病（高血压、糖尿病等）
  - 记忆力下降，需要提醒
  - 对智能手机使用有限（仅会微信基础功能）
  - 不愿给孩子添麻烦
- **目标**：
  - 独立完成日常健康管理
  - 不让子女过度担心
  - 需要时能快速获得帮助
- **痛点**：
  - 忘记吃药或吃错药
  - 出门迷路或遇到诈骗
  - 不会使用复杂的 App
  - 怕子女担心，隐瞒问题

## 五、验收标准（Acceptance Criteria）示例

### 示例 1：用药提醒（US-HEALTH-001/002/003）

**场景**：子女为父母添加降压药计划

**Given** 子女已登录并绑定父母账号  
**When** 子女进入「用药计划」页面  
**Then** 显示「添加计划」按钮

**Given** 子女点击「添加计划」  
**When** 填写药物名称「降压药」、剂量「1片」、时间「08:00」、重复「每天」  
**Then** 计划保存成功，父母端同步收到提醒

**Given** 第二天早上 8:00  
**When** 父母打开小程序  
**Then** 显示用药提醒卡片，包含药物名称、剂量、确认按钮「已服」

**Given** 父母点击「已服」  
**When** 提交成功  
**Then** 子女端实时收到通知「爸已服药，时间 08:02」

**Given** 父母未点击「已服」  
**When** 时间超过 30 分钟  
**Then** 子女收到二次提醒「爸可能忘记服药」

---

### 示例 2：扫码帮手（US-EMOTION-001/002）

**场景**：父母在餐馆不会扫码点餐，子女远程协助

**Given** 父母进入小程序「扫码帮手」页面  
**When** 点击「发起求助」  
**Then** 显示二维码扫描界面，提示「请扫描餐桌上的二维码」

**Given** 父母扫描餐桌二维码  
**When** 扫描成功  
**Then** 图片/信息自动发送到子女端

**Given** 子女收到推送通知  
**When** 点击通知查看详情  
**Then** 显示扫描到的二维码内容（如菜品列表、价格）

**Given** 子女点击「远程代点」  
**When** 选择菜品并提交订单  
**Then** 餐馆后厨收到订单，父母端显示「点餐成功，请稍等」

## 六、非功能需求（Non-Functional Requirements）

| 需求类别 | 描述 | 优先级 |
|----------|------|--------|
| **可用性** | 父母端操作不超过 3 步，字体大小可调 | 高 |
| **可靠性** | 用药提醒准时率 > 99.9%（±1 分钟） | 高 |
| **响应速度** | 子女端通知延迟 < 5 秒 | 高 |
| **隐私安全** | 位置共享需父母主动授权，且 2 小时后自动失效 | 高 |
| **兼容性** | 支持 iOS 和 Android 微信版本 8.0+ | 中 |
| **可访问性** | 支持语音播报提醒（适配视障老人） | 中 |
| **离线支持** | 用药确认可离线操作，联网后自动同步 | 中 |
| **数据备份** | 每周自动备份到云存储 | 低 |
