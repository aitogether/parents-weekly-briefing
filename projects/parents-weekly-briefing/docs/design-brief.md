# 父母周报 · 设计 Brief

> 核心目标：把 UI 和图标往三个关键词靠——**安心、温和、记得住**。

## 1. 设计关键词

| 关键词 | 含义 | 做什么 | 禁止什么 |
|--------|------|--------|----------|
| 安心 | 像靠谱的健康/家庭产品，不是花里胡哨的工具 | 圆角卡片、柔和色彩、简洁文案 | 刺眼红色、游戏化元素、炫酷动效 |
| 温和 | 颜色柔和、有留白、不堆信息 | 大量空白、浅色背景、暖色点缀 | 信息塞满每一屏、密集小图标 |
| 记得住 | 有一个简单的视觉记忆点 | 统一图标风格 + 蓝绿主色系 + 清晰布局 | 每页换风格、五颜六色 |

**禁止追求：** 复杂渐变、酷炫动效、大量图标堆砌、「设计感很强但爸妈看不懂」的美。

## 2. 优先打磨的两屏

### 2.1 子女端黄灯周报首屏（无滚动状态）

- 顶部状态卡 + 一句话总结 + 1 条最关键的事实 → 一屏内全部看清
- 色彩层次分明：背景、卡片、状态灯、按钮各有明确层级
- 图标风格统一（线性、少量、不抢文案戏份）

### 2.2 演示 App 首页（三个大按钮菜单页）

- 三个选项像「三张大卡片」，而不是普通按钮
- 每个卡片：小图标 + 简短标题 + 一句副标题
- 颜色柔和、有留白，不要挤成一团

**验收标准：** 打磨到「拿给非技术的人看，第一句是'看起来挺舒服的'而不是'好复杂/好专业啊'」。

## 3. 图标规范

### 3.1 App 主图标（Launcher Icon）

- 极简图形：小房子轮廓 + 心形/心跳线，或 2 个简化人形（父母）+ 1 个小点（子女）
- 色板：只用 2–3 种颜色（主蓝绿 + 一点点暖色）
- 风格：扁平、线性、无复杂渐变

### 3.2 状态/功能图标（页面内）

- 统一使用 Material Symbols Outlined 一套为基础
- 固定映射：
  - 步数 → 脚印/鞋子轮廓
  - 用药 → 药丸/药瓶
  - 状态灯 → 简化圆点/信号灯
  - 回声 → 对话气泡
- 颜色：尽量单色线条（深蓝灰），只在状态灯上用彩色

### 3.3 数量控制

- 每个页面最多 3–4 个图标
- 如果犹豫要不要加图标 → 先不加

## 4. UI 细节改进

### 4.1 字体和字号

**子女端：**
- 状态标题：18–20
- 正文文案：14–16，行距拉开

**父母端：**
- 大按钮文字 ≥ 18
- 按钮高度明显高于其他控件

### 4.2 卡片阴影和边框

- 不用重阴影，改用轻微阴影或浅边框
- 背景：浅灰
- 卡片：纯白 + 1px 超淡边或极浅阴影

### 4.3 间距

- 卡片上下 ≥ 1 行文字高度的空隙
- 按钮和其他元素之间 ≥ 一个拇指宽
- **宁可看起来简洁甚至有点空，也不要塞满**

## 5. 验收方法

### 5.1 用户测试（找一个非技术朋友）

给他看两屏（子女端黄灯周报首屏 + 父母端用药页），问三句话：
1. 「你第一眼觉得这东西是干嘛的？」
2. 「看起来有点暖，还是有点紧张？」
3. 「有没有哪一块你觉得太花/太吓人？」

### 5.2 自己主观感受

把两屏截图放在朋友圈/推文里，还觉得「丑到不想发」吗？
如果不丑、甚至有点想发 → 已达到「眼前一亮」的及格线。

## 6. 一句话 Sprint 目标

> 功能先别动了，接下来一个 Sprint 只干一件事：把「子女端黄灯周报首屏」和「演示 App 首页」打磨到让我愿意发朋友圈。图标统一线性、颜色统一蓝绿系、间距留白更舒服，宁可简单，不要炫技。
